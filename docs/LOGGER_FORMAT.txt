PURSUIT OF PEACE — SESSION LOG FORMAT
======================================

Log files are written to logs/session_<YYYYMMDD_HHMMSS>.log
One file is created per game session.

Each line is a structured log entry with the following tab-separated format:

  [TIMESTAMP]  [LEVEL]  [MESSAGE]  [CONTEXT?]

Example entries:
  2026-04-08 14:08:55  SYSTEM   App initialised.
  2026-04-08 14:09:01  INFO     Player navigated to: tavern
  2026-04-08 14:09:04  DATA     rest | {"gold_spent": 10}
  2026-04-08 14:09:10  DATA     combat_start | enemy_bloat_rat_01
  2026-04-08 14:09:12  DATA     combat_attack | {"damage": 6, "remaining_hp": 4}
  2026-04-08 14:09:15  DATA     combat_flee
  2026-04-08 14:09:20  WARN     Lore key not found: 'lore_room_empty_01'  [DungeonManager.get_room_lore_text]
  2026-04-08 14:09:30  ERROR    Failed to load profile Jack: Missing player_state in profile


LOG LEVELS
----------

  SYSTEM  — Application lifecycle events (startup, shutdown, data load).
  INFO    — Player navigation and significant game transitions.
  DATA    — Game event data (gold spent, items taken, combat results).
            Format: event_id | payload (JSON or plain string)
  WARN    — Recoverable problems (missing file, unknown key, fallback used).
            May include a [context] suffix showing the function name.
  ERROR   — Failures that may degrade gameplay (failed saves, corrupt data).


AUDIT TOOL
----------

Use audit.pyw to browse log files with a GUI:

  python audit.pyw

The audit tool supports:
  - Selecting a log file from the session list.
  - Filtering by log level.
  - Searching by keyword.
  - Viewing raw log text.
